Impacts

The Rubber Ducky relies on the fact that, at some level, user input must be trusted. At a computer without this trust, no one would ever be able to do anything, so it would be useless. This means security is necessarily limited, and the Rubber Ducky exploits this by masquerading as user input. The impact of this is that security now requires mechanisms to verify user input, or otherwise prevent malicious input from ever entering the system. Antivirus software, firewalls, and network management are all bypassed, so another layer of security becomes necesary. There are three main areas that are impacted: trusting user input, trusting devices, and the imbalance between attacker and defender.

Trusting User Input

The Rubber Ducky, if programmed well, is very difficult to distinguish from user input. Effectively, anything a user could legitimately do or gain access to, the Ducky can as well. A typical exploitation of a computer starts with some security flaw and builds up to gaining control of a user account. The Ducky bypasses all of this and gives the attacker direct control of the user account, temporarily. Thus, the only limits on the damage are limits that apply to normal users
